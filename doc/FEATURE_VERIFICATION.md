# Feature Verification Checklist

This document verifies each requirement from **project description**, **doc/frontend/frontend for gamefowl derby.md**, and **doc/frontend/Derby Attribute.md** against the current codebase. ✅ = fully implemented. ⚠️ = partially implemented or optional. ❌ = not implemented.

---

## 1. Project Description (Original Spec)

| Requirement | Status | Verification |
|-------------|--------|---------------|
| Browser-based, unlimited entrants | ✅ | React SPA; no entry cap; `tournament_engine` pairs any count (bye when odd). |
| Single bracket per entrant, 10–20 rounds configurable | ✅ | Admin create: `total_rounds` 10–20; one entry per user per tournament. |
| Winners advance; pair only winners vs winners | ✅ | `get_entries_for_round`: round 1 = all active, round N = winners from N−1. |
| Eliminated entries get no further pairings | ✅ | `mark_eliminated()` + `get_entries_for_round()` filter by status and previous-round winners. |
| Tie-breakers until one undisputed winner | ✅ | `advance_tournament_round`: after last scheduled round, `is_tie_breaker` runs until single winner. |
| SSO; no local passwords; autogenerated username | ✅ | Google OAuth: `GET /api/auth/google`, `POST /api/auth/google/token`; SSO users have `hashed_password=None`; username `player_xxxx`. Email/password still available when Google not configured. |
| Admin: tournament name, number of rounds (10–20), start date/time | ✅ | Admin create form: name, derby type, rounds, **datetime-local** start, **prize tier** (Standard/Grand/Prestigious). |
| Real-time bracket or match list | ✅ | WebSocket `ws/tournaments/{id}`; Bracket page subscribes and refetches on message. |
| Restart or clone tournament without code change | ✅ | Admin: Restart (clear matches, reset entries), Clone (new tournament same config + prize_tier). |
| Source code and build instructions | ✅ | README: backend (uvicorn), frontend (npm run dev / build). |
| Quick user guide (sign-up, admin, tie-breakers) | ✅ | README “User guide (short)” section. |
| No browser plugins; Chrome, Firefox, Safari, Edge | ✅ | Standard web app; no plugins. |

**Intentional / environment:** Deployed version is not in repo (run locally or deploy per README). SSO “basic security tests” (token expiration, CSRF) depend on provider and deployment (HTTPS, cookies).
